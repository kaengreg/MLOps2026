MVP-пайплайн для задачи предсказания стоимости поездки (`total_amount`) на датасете **NYC Yellow Taxi Trip Data**.

Проект реализует минимально рабочую ML-систему для потоковой обработки табличных данных:
- чтение исходного датасета;
- разбиение данных на временные батчи;
- проверку качества данных;
- подготовку признаков;
- обучение нескольких моделей;
- небольшой подбор гиперпараметров на этапе обучения;
- обновление моделей на новых батчах;
- сохранение versioned-артефактов и текущих версий моделей;
- инференс на новых данных;
- сбор итогового markdown-отчета по качеству данных и моделей.

---

## Цель проекта

Предсказать `total_amount` по характеристикам поездки.

В качестве признаков используются только те поля, которые доступны без раскрытия целевой переменной. Компоненты итоговой цены и сам `total_amount` исключаются из обучения, т.к. иначе модель может научиться выстраивать `total_amount` по ним.

## Как запускать этапы

Ниже приведен рекомендуемый порядок запуска этапов.

```bash
chmod +x data_loading.sh && ./data_loading.sh 
python run.py --mode prepare_data
python run.py --mode train
python run.py --mode inference --file ./path_to_file.csv
python run.py --mode update
python run.py --mode summary
```

### 1. Подготовка данных

Команда:
```bash
python run.py --mode prepare_data
```

Что делает:
- читает исходный датасет NYC Yellow Taxi Trip Data;
- для обучения использовался `yellow_tripdata_2015-01.csv`;
- сортирует записи по времени;
- разбивает `yellow_tripdata_2015-01.csv` на последовательные временные батчи;
- сохраняет обучающие батчи в `data/raw_batches/`.

Когда запускать:
- перед первым обучением;
- при изменении исходного датасета;
- если нужно заново пересобрать поток батчей.

### 2. Обучение моделей

Команда:
```bash
python run.py --mode train
```

Что делает:
- читает подготовленные батчи;
- строит train/test разбиение по батчам;
- обучает несколько моделей (`CatBoostRegressor`, `DecisionTreeRegressor`, `MLPRegressor`);
- выполняет небольшой подбор гиперпараметров;
- выбирает лучшую модель по качеству;
- сохраняет current-версии всех моделей и отдельную best-модель.

Когда запускать:
- после `prepare_data`;
- после полной пересборки данных;
- когда нужно переобучить систему с нуля.

### 3. Инференс на внешних данных

Команда:
```bash
python run.py --mode inference --file ./path_to_file.csv
```

Что делает:
- загружает текущую лучшую модель;
- использует внешние данные, подготовленные из `yellow_tripdata_2016-01.csv` (или любого другого при желании);
- подготавливает признаки для внешнего файла или внешнего батча;
- строит предсказания;
- сохраняет CSV с дополнительной колонкой `predict`;
- если во входном файле есть целевая переменная, дополнительно считает метрики инференса.

Когда запускать:
- для применения лучшей обученной модели к новым данным;
- для локальной проверки качества на внешнем файле.

### 4. Обновление моделей на новом батче

Команда:
```bash
python run.py --mode update
```

Что делает:
- берет следующий необработанный батч из потока;
- обучающий поток формируется из батчей в `data/raw_batches/`, которые не были обработаны в train;
- считает метрики качества данных для нового батча;
- обновляет все модели-кандидаты;
- для `CatBoostRegressor` выполняет дообучение на новом батче;
- для `MLPRegressor` выполняет `partial_fit` на новом батче;
- для `DecisionTreeRegressor` выполняет полное переобучение на накопленных батчах с сохраненными гиперпараметрами;
- сравнивает обновленные модели и выбирает лучшую;
- обновляет current-артефакты моделей и best-модель.

Когда запускать:
- после появления нового батча;
- для эмуляции потокового обновления системы.

### 5. Построение итогового отчета

Команда:
```bash
python run.py --mode summary
```

Что делает:
- собирает данные из `data_quality_log.csv`, `training_log.csv`, `update_log.csv`;
- читает metadata текущей лучшей модели;
- формирует итоговый markdown-отчет `reports/summary_report.md`.

Когда запускать:
- после обучения;
- после нескольких update-шагов;
- перед сдачей или демонстрацией проекта.

## Примечания по моделям

В проекте используются три модели:
- `CatBoostRegressor`;
- `DecisionTreeRegressor`;
- `MLPRegressor`.

На этапе `train` для моделей выполняется небольшой подбор гиперпараметров.
На этапе `update` найденные гиперпараметры переиспользуются:
- `CatBoostRegressor` дообучается на новом батче;
- `MLPRegressor` обновляется через `partial_fit`;
- `DecisionTreeRegressor` переобучается полностью, так как обычное дерево решений не поддерживает инкрементальное обучение.

## Используемые части датасета

В текущем MVP данные разделены по месяцам:
- `yellow_tripdata_2015-01.csv` используется как обучающая часть и разбивается на батчи в `data/raw_batches/`;
- `yellow_tripdata_2016-01.csv` используется как внешняя часть и может быть разбита на батчи в `data/external/` для инференса и отдельных проверок.
